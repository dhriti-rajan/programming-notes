Lesson 1: The Basics of the Web and HTMLBack to Top ˆ

Introduction to the WebBack to Top ˆ

HTML is the main document type of the web. The web runs on HTTP, which is the protocol that unites it.

World-wide Web:Back to Top ˆ

The World-wide web is a collection of HTML documents, or, documents made of HTML(HyperText Markup Language), and was invented in the 1990's. HTML documents are the basis for almost every web page. These documents are accessed by URL's. A URL looks something like this:

http://www.fancyicons.com/free-icons/220/foods/png/256/ice_cream_256.png

It refers to a certain document on the web. When a person goes to a certain web page, their computer sends a HTTP Request to a server. The server then sends back the corresponding HTML document where it is displayed on the screen by a web browser(chrome, safari, firefox, etc.)

Components of the Web:Back to Top ˆ

The only things that the web needs to run are you, your computer running a browser, the internet, and servers. Servers are also computers, except they host files, rather than browsing them.

HTML BasicsBack to Top ˆ

HTML is made up of multiple things. One of them is text content. Text content is what you see on a HTML document. Another component that makes up HTML is markup. Markup is what the text content looks like. HTML is also made up of references to other documents. These can be images, videos, or many other things. The last thing that HTML is made of is links to other pages.

If you type plain text in HTML, it ends up looking like plain text. To make text look different, we need to use markup.

HTML TagsBack to Top ˆ

Markup is made of things called tags, which make text look different. Tags have one main structure. Unless the tag is a void tag, the opening tag comes first, followed by the contents, and ended with the closing tag. A void tag has does not have a closing tag, and therefore does not need content. As a result, void tags are composed only of opening tags.

There are multiple types of tags. Some of them are the bold<b>, italic;<em>, span<span>, div<div>, break<br>, paragraph<p>, anchor<a>, and image<img> tags. Of all of these tags, the only void tag is the break tag. Some of these tags have the same function, but do this in different ways. For example, the break tag and the paragraph tag both put text on multiple lines. However, they are different because they are different types of tags. The break tag is an inline tag, whereas the paragraph tag is a block tag. Inline tags are only text, but block tags make an imaginary box around the text. This box can have a certain height or width.

Text ends up in one long line unless you use either the break tag, or the paragraph tag. We can use something called whitespace to make text appear on seperate lines. This is just using the break or paragraph tag multiple times.

Tags can have multiple attributes. Attributes are extra bits of information. They are used to define the characteristics of an HTML element and are placed in the opening tag.

Why Computers Are StupidBack to Top ˆ

Some programmers describe computers as "stupid". Usually, what they mean is that computers are very sensitive to typos and small mistakes, whereas humans can still understand it. Because they are so sensitive, what we consider "small mistakes" could lead to big problems in programming. For example, if you forget to put an end tag but remember the opening tag, the computer applies the function of the opening tag to all the text following it instead of the text section you meant to apply it to.

Understanding CSSBack to Top ˆ

CSS stands for "Cascading Style Sheets." You can have multiple CSS files or sheets that will be referenced in HTML. All of these will then be used to find the final and visible style for your page.

[ND] Thinking About CascadingBack to Top ˆ

  • Inheritance is a key feature in CSS
  • Inheritance relies on the ancestor-descendant relationship to operate
  • Inheritance - the mechanism by which properties are applied not only to a specific element, but also to its descendants
  • box-related properties are not inherited
  • properties that can be inherited: color, font, letter-spacing, line-height, list-style, text-align, text-indent, text-transform, visibility, white-space, and word-spacing
  • properties that can’t be inherited: background, border, display, float and clear, height and width, margin, min- and max-height and -width, outline, overflow, padding, position, text-decoration, vertical-align, and z-index
  • Inheritance prevents repetition in a style sheet, so it is easier to write/
  • inheritance enhances faster-loading of web pages by users and enables the clients to save bandwidth and development costs
  • descendant elements inherit text-related properties
  • descendant elements may inherit CSS property values from any ancestor element enclosing them
  • inheritance relies on the document tree, which is the hierarchy of (X)HTML elements in a page based on nesting

[ND] CSS Summary and ReferenceBack to Top ˆ

What is CSS?Back to Top ˆ

CSS is code written to make HTML elements look different.

How Does CSS "Control" HTML?Back to Top ˆ

CSS controls how HTML elements look. For example: If you want to write CSS that makes all h3 elements have a red background and a font size of 20 pixels, you would write:

      h3 {
      background-color: red;
      font-size: 20px;
      }

In this example, the h3 is a selector. In other words, it tells the browser,"Apply the following rules to all of the h3 elements."

After the h3 there is a left flower bracket ({) and at the bottom there is a matching right flower bracket (}). Everything between these flower brackets is a rule that the browser will read and follow.

The line of code that says background-color : red; is a declaration. background-color is a property and in this case, red is the value of that property.

CSS VocabularyBack to Top ˆ

  • CSS = Cascading Style Sheets
  • Style: this word can refer to many things, such as:
  • The HTML element. For example: <style>div {color: blue;}</style>
  • The HTML attribute. For example:<div style="color : blue">this text would be blue</div>
  • the overall look of a web page
  • a verb: to style, meaning, to make something look different
  • rule - a line of CSS code that describes the value that a specific attribute should have
  • property - what you want to change
  • value - the value that you want a certain attribute to have
  • selector - the name that is used to target the elements assigned to a specific class or id attribute in the HTML
  • class - a class refers to a certain group of elements that can be styled together. Class names cannot contain any periods or any other types of punctuation marks
  • id's - an id is an unique way to identify elements in HTML

Selecting by ClassBack to Top ˆ

In the previous example, we set the font-size of every h3 to 20px. If we only want to add this style to a few of the h3's, we have to use class selectors. To do this, we can add the same class to each of these specific h3's using the div tag. For example: Say we wanted to make the font size of the h3's "Hello" and "Goodbye" 20 pixels. We could write:

      <div class="three">Hello</div>
      <div class="three">Goodbye</div>

      <style>
      .three {
        font-size: 20px;
      }
      </style>

We can also apply different rules to certain bits of text. For example: Say we wanted to make the background-color of the h3 "Hello" red, and the background-color of the h3 "Goodbye" blue. We could write:

      <div class="1">Hello</div>
      <div class="2">Goodbye</div>

      <style>
      .1 {
        background-color: red;
      }
      .2 {
        Background-color: blue;
      }
      </style>

In both of these examples, we refer to an HTML element's class-name as .class-name to add style to it.

How Do I Include CSS Styling in My Web Page? Back to Top ˆ

There are three ways in which you can include CSS styling in your web page.

The first way is to write CSS in the <head> of your HTML. This method is great for really small projects. To do this:

  1. Add a head element at the top of your HTML document
  2. Add a style element inside the head element
  3. Write your CSS inside the style element and put your HTML in the body below

For example, we could write:

      <head>
        <style>
          div {
            background-color: red;
          }
        </style>
      </head>
      <body>
        <div>
          hello
        </div>
      </body>

The second way is to link your HTML to a separate CSS file. This adds another step, but it lets you stay more organized when working on larger projects. To do this:

  1. Write all of your structural HTML in one file which we can call main.html
  2. Put all of your CSS code in a separate file which we can call main.css
  3. Add a <head> element to the top of your HTML
  4. Add a link to your CSS file in the head
  5. Add the following attributes to the link: rel="stylesheets", type="text/css", href="main.css"

The third way to add CSS styling to your page is to write it inline with your HTML. Usually this is a bad idea since you have to repeat lots of code. However, it is a good idea to be familiar with this method since you may see code that uses it. Using this method, you modify the style attributes of each individual HTML element. For example: Say we want to make the word "hello" have a background-color of turquoise. We could write:

      <div style="background-color: turquoise;">
        hello
      </div>

We can also give bits of text different colors. For example: if we want the word "hello" to have a background-color of red and the word "goodbye" to have a background-color of blue, we could write:

      <div style="background-color: red;">
        hello
      </div>
      <div style="background-color: blue;">
        goodbye
      </div>

Note: you can make many mistakes with all of these methods. For example, method two will not work if you have the main.css and main.html files in separate folders.

Box SizingBack to Top ˆ

There are four main points about box sizing:

  1. HTML elements are boxes and each box has 4 components
  2. Because there are so many components to each box, it can often be hard to get the size of a box just right
  3. There are two techniques you can use to help deal with sizing issues:
    1. Set sizes in terms of percentages rather than pixels
    2. Set the box-sizing attribute to border-box for all of the elements
  4. Different browsers work slightly differently which sometimes causes them to display the same code in different ways
Box PositioningBack to Top ˆ
  1. Divs are block elements (as opposed to inline), so by default they take up the entire width of a page
  2. Adding the rule display: flex; to the appropriate CSS will override this behavior and let divs appear next to each other
